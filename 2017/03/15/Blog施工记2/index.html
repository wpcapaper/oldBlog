menu - 自定义页面及菜单，依照已有格式填写。填写后请在source目录下建立相应名称的文件夹，并包含index.md文件，以正确显示页面。导航菜单中集成了FontAwesome图标字体，可以在这里选择新的图标，并按照相关说明使用。

到这里我们有两个选择：1、去掉about 2、加上about页面；既然这篇文章在这放着，我们肯定选1(
如何添加页面，文档里也有提到：

添加页面
在source目录下建立相应名称的文件夹，然后在文件夹中建立index.md文件，并在index.md的front-matter中设置layout为layout: page。若需要单栏页面，就将layout设置为 layout: single-column。

那么问题来了，source目录是皮肤里的source还是根目录的source？啥是front-matter？
首先我们回答第一个问题，source是根目录里的source目录，当然你也可以试试放到皮肤的source目录里看看会发生什么(

再查阅一下Hexo的文档，我们可以确认前面这个用”—“分割开的就是Front-matter了，具体的一些设定选项我也搬了过来，注意hexo的表格写法：

其中|:—–|表示左对齐，|:—–:|表示居中，|—–:|为右对齐

Setting Description Default
layout Layout
title Title
date Published date File created date
updated Updated date File updated date
comments Enables comment feature for the post true
tags Tags (Not available for pages)
categories Categories (Not available for pages)
permalink Overrides the default permalink of the post
